## Findings

- CRITICAL BUG: Production build's OpenAPI document (getOpenAPIDocument) only includes /todos paths
  - This means autoRegisterRealtime cannot discover WS/SSE routes
  - hasWSPath('/api/chat/ws') returns false in production
  - WebSocket upgrade requests are rejected (socket.destroy())
- WORKAROUND: Manually register WS paths in node.ts before createApp()
- The chat API (/api/chat/ws/status) works fine in production - only the WS upgrade is affected
- Vite dev server on port 5176 (or any available port) also had WS issues - the websocketPlugin may not receive upgrade events properly
- Node.js test client confirms WS works when paths are manually registered
