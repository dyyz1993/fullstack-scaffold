# Bug #001

## 基本信息

- **发现时间**: 2026-05-09
- **模块**: content
- **严重程度**: P1-严重
- **状态**: 待修复

## 复现步骤

1. Navigate to http://localhost:5175/content
2. Page shows "暂无内容" empty state
3. Check API: GET /api/public/contents returns `{ success: true, data: { items: [...], count: 7 } }`
4. ContentListPage.tsx line 37 treats `data` as array: `setContents(Array.isArray(data) ? data : [])`
5. Since `data` is `{ items: [...], count: 7 }` (not an array), `Array.isArray(data)` is false
6. Result: `setContents([])` — always empty

## 预期行为

Content list page should display 7 content cards from the API.

## 实际行为

Content list page shows "暂无内容" (empty state) despite API returning 7 items.

## 根因分析

API response shape is `{ success: true, data: { items: [...], count: 7 } }` but `ContentListPage.tsx` expects `data` to be a direct array.

## 修复建议

In ContentListPage.tsx line 37, change:

```typescript
// Before:
setContents(Array.isArray(data) ? data : [])

// After:
setContents(Array.isArray(data) ? data : Array.isArray(data?.items) ? data.items : [])
```
