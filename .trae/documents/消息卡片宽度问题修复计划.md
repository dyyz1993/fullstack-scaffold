# 消息卡片宽度问题调查与修复计划

## 问题分析

### 根本原因
在 [MessageCard.tsx](file:///Users/xuyingzhou/Project/create-biomimic-app/template/src/client/components/MessageCard.tsx#L38-L42) 中，消息卡片的根容器没有设置宽度限制：

```jsx
<div
  className={`p-4 bg-white rounded-lg border-l-4 shadow-sm ${className}`}
  style={{ borderLeftColor: borderColor || undefined }}
  data-testid="message-card"
>
```

虽然内部的 `<pre>` 标签有 `whitespace-pre-wrap break-words` 来处理内容换行，但外层容器没有 `max-width` 限制，导致卡片在内容超出时会无限扩展。

### 影响范围
- **MessageCard 组件**：用于 WebSocket 页面显示各种消息类型（ping、echo、broadcast 等）
- 当 JSON payload 包含长字符串且没有空格时，卡片会溢出容器

---

## 修复方案

### 步骤 1：修改 MessageCard.tsx
在卡片根容器添加 `max-w-full` 或 `w-full` 来限制宽度：

**修改位置**: [MessageCard.tsx#L38-L42](file:///Users/xuyingzhou/Project/create-biomimic-app/template/src/client/components/MessageCard.tsx#L38-L42)

**修改内容**:
```jsx
// 修改前
className={`p-4 bg-white rounded-lg border-l-4 shadow-sm ${className}`}

// 修改后
className={`p-4 bg-white rounded-lg border-l-4 shadow-sm max-w-full ${className}`}
```

---

## 验证方法

1. 检查 MessageCard 组件的 CSS 类是否包含 `max-w-full`
2. 重新渲染包含长字符串的 JSON payload 的消息卡片
3. 确认卡片不会超出父容器
